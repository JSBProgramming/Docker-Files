I'll walk you through starting and finishing a Python project with Docker from the command line.
Starting the Project
1. Create project structure:
bashmkdir my-python-project
cd my-python-project
mkdir app
touch app/__init__.py app/main.py
touch requirements.txt Dockerfile .dockerignore
2. Write your Python application (app/main.py):
bashcat > app/main.py << 'EOF'
def main():
    print("Hello from Docker!")

if __name__ == "__main__":
    main()
EOF
3. Add dependencies (requirements.txt):
bashecho "requests==2.31.0" > requirements.txt
4. Create Dockerfile:
bashcat > Dockerfile << 'EOF'
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY app/ .
CMD ["python", "main.py"]
EOF
5. Create .dockerignore:
bashcat > .dockerignore << 'EOF'
__pycache__
*.pyc
*.pyo
*.pyd
.Python
env/
venv/
.git
.gitignore
EOF
Building and Running
Build the Docker image:
bashdocker build -t my-python-app:latest .
Run the container:
bashdocker run --rm my-python-app:latest
Run interactively (for development):
bashdocker run -it --rm -v $(pwd)/app:/app my-python-app:latest bash
Run with port mapping (if it's a web app):
bashdocker run -p 8000:8000 --rm my-python-app:latest
Finishing/Cleanup
Stop running containers:
bashdocker ps  # list running containers
docker stop <container_id>
Remove the image:
bashdocker rmi my-python-app:latest
Clean up all unused Docker resources:
bashdocker system prune -a
Export image (to share or archive):
bashdocker save my-python-app:latest | gzip > my-python-app.tar.gz
This gets you from zero to a containerized Python application. The key is having a clear Dockerfile that defines your environment, then using docker build to create the image and docker run to execute it.
